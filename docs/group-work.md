# グループ作業の進め方

## 第1回の授業内

1. 全員でプログラムを実行する
2. 現在の仕様とテストを読む
3. 問題箇所を洗い出す
4. 優先順位を付ける
5. 最初のIssueを作る
6. 作業単位とレビュー担当を決める

## 授業間

- 全員が少なくとも1件のIssue，実装，テスト追加，レビューのいずれかに責任を持つ
- 実装担当とレビュー担当を固定せず，途中で交代する
- 設計方針を変えた場合は，IssueまたはPRに理由を残す
- コミットメッセージから，変更の順序を追えるようにする
- Pythonの言語機能を使ったこと自体ではなく，変更範囲や不正状態への効果を説明する

## 第2回の発表準備

発表資料には，コード断片を大量に貼るより，次を優先してください．

- 問題と考えた構造
- 変更時に起こる困難
- 責務や依存関係の変更
- 変更前後の図
- テストとコミットの流れ
- トレードオフ
